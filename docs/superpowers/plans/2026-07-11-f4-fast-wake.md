# F4 — Wake detection rápida (tiny dedicado) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Bajar la latencia de detección de "escúchame kiki" de ~2-3s a <1s usando un Whisper tiny (~75MB) residente dedicado a verificar la frase, sin degradar la calidad del dictado same-breath.

**Architecture:** `WhisperTranscriber` se vuelve parametrizable por modelo (`init(model:)`); `AppDelegate` crea una segunda instancia con `openai_whisper-tiny` y se la inyecta a `WakeListener` vía un nuevo seam `setWakeVerifier(_:)` cuando termina de cargar. `WakeListener.handleListeningSegment` verifica con el tiny; si el match trae remainder (dictado en el mismo aliento), re-transcribe ese segmento con el modelo grande antes de entregarlo — el tiny solo decide *si* la frase se dijo, nunca produce texto que se inserte.

**Tech Stack:** Swift 5.10, WhisperKit v1.0, XCTest.

## Global Constraints

- Spec: `docs/superpowers/specs/2026-07-11-cuatro-features-v1-design.md` §F4
- Modelo wake: `"openai_whisper-tiny"` (multilingüe — NUNCA `tiny.en`, la frase ES es primaria)
- El texto que se inserta como dictado JAMÁS proviene del tiny (regla de calidad)
- Fallback: si el tiny no carga, `WakeListener` sigue verificando con el transcriber principal (comportamiento actual — degradación, no rotura)
- `endSilence` de listening queda en 0.5s (spec: no se toca en esta feature)
- Concurrencia de `WakeListener`: TODO estado mutable confinado a `queue` (leer la doc "Disciplina de concurrencia" del archivo antes de tocar nada); eventos al delegate con el patrón `notify`/`notifyCapture` existente; fence de sesión (`capturedSession == self.session`) en TODA tarea async nueva
- Logs con `KikiLog` incluyendo timings; regla de privacidad existente: NO loggear contenido de segmentos sin match
- Tests: `swift test`; app target con `make build` (xcodebuild); NO correr `make run`

1. Nueva propiedad confinada a `queue` + setter público (mismo patrón `queue.sync` de los métodos públicos existentes):

3. `applyMatch` cambia de firma: `applyMatch(_ text: String, language: String, samples: [Float], usedVerifier: Bool)` (el call-site ya tiene los samples; `sampleCount` se deriva). El caso remainder-vacío (`arm()`) no cambia. El caso remainder NO vacío:
   - `usedVerifier == false` → comportamiento actual exacto (notifyCapture sameBreath con el remainder).
   - `usedVerifier == true` → NO entregar el remainder del tiny. Lanzar la re-verificación con el modelo principal (nueva función, misma disciplina que la Task de `handleListeningSegment`: fence de sesión con `capturedSession`, `queue.async` para volver, log de timings SIN contenido hasta confirmar match):
```swift
    /// F4: el tiny detectó frase+remainder en el mismo aliento. Su texto no
    /// tiene calidad de dictado, así que el segmento completo se
    /// re-transcribe con el transcriber principal y se entrega SU remainder.
    /// Si el principal no reconoce la frase (transcribió distinto), se
    /// entrega su texto completo: el tiny ya estableció que el usuario se
    /// dirigía a kiki, y perder dictado es peor que un prefijo imperfecto.
    private func reverifySameBreath(_ samples: [Float]) {
        dispatchPrecondition(condition: .onQueue(queue))
        let transcriber = self.transcriber
        let capturedSession = session
        Task {
            var detectedLanguage = "es"
            let started = Date()
            let text: String?
            do {
                text = try await transcriber.transcribe(samples)
                if let languageDetecting = transcriber as? LanguageDetecting {
                    detectedLanguage = await languageDetecting.detectedLanguage()
                }
            } catch {
                KikiLog.log("kiki wake: re-verificación same-breath falló (\(error))")
                text = nil
            }
            let seconds = Date().timeIntervalSince(started)
            self.queue.async {
                guard capturedSession == self.session else { return }
                guard let text, !text.isEmpty else { return }
                KikiLog.log("kiki wake: same-breath re-verificado en \(String(format: "%.1f", seconds))s")
                let delivered = WakePhraseMatcher.match(text)?.remainder ?? text
                let language = detectedLanguage
                self.notifyCapture {
                    $0.wakeListenerDidCaptureSameBreath(
                        text: delivered, language: language, sessionIsCurrent: $1)
                }
            }
        }
    }
```
   Nota de estado: revisar qué hace hoy el path same-breath con `_state`/sesión inmediatamente después de `notifyCapture` (si arma, para o queda listening) y replicar EXACTAMENTE esa transición en el camino re-verificado — la re-verificación cambia *qué texto* se entrega, no la máquina de estados. Si la transición actual ocurre dentro de `applyMatch`, ejecutarla al despachar `reverifySameBreath` (no al completarse), para que el mic/HUD reaccione de inmediato.

**Resultado del experimento 2026-07-11 (`.superpowers/sdd/wake-experiment-results.md`):** tiny SIN bias solo matchea 1/4 fixtures ("SKHAIM KIKI"); tiny CON initial-prompt bias (términos "escúchame", "kiki", "listen to me kiki" vía `DictionaryProviding`) matchea **4/4 positivos y rechaza el control negativo**, latencia ~0.35s. `base` es peor (2/4, y con bias pierde la frase en same-breath). /// Términos de sesgo (initial prompt de Whisper) para el verificador tiny de
/// la frase de activación (F4). Sin este bias, tiny transcribe la frase ES
/// de forma irreconocible ("SKHAIM KIKI" — experimento 2026-07-11, ver

}
